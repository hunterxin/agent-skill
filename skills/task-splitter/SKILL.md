# task-splitter

## 目标
根据用户给出的需求描述，将任务拆分为多个逻辑阶段（phase），每个阶段包含明确的步骤和输出。逐步引导用户完成整个任务。

## 工作流程

1. **理解需求** – 阅读用户的输入，识别核心目标、主要模块/功能、可能的难点。
2. **拆分阶段** – 将任务分解为3~6个阶段，每个阶段有：
   - 阶段名称
   - 阶段目标
   - 具体步骤（1~N）
   - 期望输出 / 验收标准
3. **输出计划** – 以结构化方式返回阶段列表。
4. **按阶段引导** – 当用户准备好，逐一执行每个阶段。每完成一个阶段，要求用户确认再进入下一阶段。

## 输出格式

最终输出一个 JSON 对象，包含：
```json
{
  "task": "<任务名称>",
  "phases": [
    {
      "name": "<阶段名>",
      "goal": "<阶段目标>",
      "steps": ["步骤1", "步骤2", ...],
      "output": "<期望输出>"
    }
  ]
}
```
